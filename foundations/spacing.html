Overview

The spacing system uses a compact scale built around 4px increments. The same tokens are reused for field padding, panel insets, section rhythm, and vertical spacing between component groups.

Use spacing by role rather than by component. Insets define internal breathing room, gaps define relationships between adjacent items, and stack spacing controls the rhythm between larger blocks.

Spacing Principles

Use spacing by role, not by component. Prefer consistent steps over one-off values. Use gap for relationships and padding for containment.

Patterns

These patterns show how the spacing scale should be applied inside components and between related items.

Insets

Padding inside components

Use smaller steps for compact controls and larger steps for structural containers.

  • Fields Horizontal padding uses --space-3. Vertical padding uses --space-2.
  • Panels Horizontal insets use --space-6. Panel body top padding uses --space-4.
  • Sections Use inset values aligned with panel rhythm to maintain consistency across larger containers.

Gaps

Spacing between related items

Use gaps to express relationships, not structure.

  • Small internal relationships Use --space-2 and --space-3 for labels, choices, and utility controls.
  • Content groups Use --space-5 for stacked fields and section content.
  • Container stacks Use --space-8 for panel-to-panel spacing and larger grouped content.

Padding vs Gap

Choosing the right spacing tool.

  • Padding Defines internal space within a field, card, panel, or section.
  • Gap Defines relationships between adjacent elements.
  • Margin Avoid for layout. Use only for page-level exceptions.

Rhythm

These ranges define the vertical spacing patterns used across the system.

Tight rhythm: 8px – 12px
Standard rhythm: 16px – 20px
Section rhythm: 24px – 32px

Panels

Wide horizontal insets with tight vertical rhythm

  • Header inset--panel-header-padding-x = --space-6.
  • Body inset--panel-body-padding-x = --space-6.
  • Body top / bottom--space-4 top and --space-8 bottom.

Sections

Content rhythm inside larger containers

  • Section spacing after--section-spacing-after = --space-5.
  • Content gap--section-content-gap = --space-5.
  • Related content--section-related-padding-top = --space-5 and --section-related-gap = --space-3.

Cards and Rows

Dense information surfaces

  • Record row spacingRows use compact fixed paddings and a small internal content gap to stay dense and scannable.
  • Card spacingCards use small internal gaps and tighter padding than panels so lists can stay compact.
  • System directionDense surfaces stay tighter than forms and panels, but still pull from the same 4px-based rhythm.
