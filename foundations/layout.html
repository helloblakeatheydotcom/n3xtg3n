Layout

Layout defines the shared structural rules used across the system: the 12-column base grid, 16px gutters, container widths, responsive breakpoint behavior, and supported field spans. Components and patterns build on this foundation.

Overview

Layout defines the shared mechanics of the system. It sets how content aligns to the base grid, how gutters are applied, how wide panel-contained forms may grow, and how widths are expressed as field spans. Pattern pages decide when to use specific compositions.

Base Structure

12-column foundationAll structural placement maps back to a 12-column grid, even when the visible grouping simplifies at smaller widths.

16px guttersColumns are separated by a consistent 16px gutter so spacing remains stable across components and patterns.

Container width is layout-controlledPanels may grow to full width, while layout constrains common panel-contained form surfaces to the approved maximum width.

Responsive Capability

Viewport width changes visual groupingThe same base grid can present broader or narrower groupings as available width changes.

Breakpoints define capacityThe system supports wider, medium, and narrow presentations without redefining the underlying grid.

Patterns choose compositionWhether a form is presented as one column, paired columns, or denser grouped rows is a pattern decision, not a layout rule.

Nested Contexts

Nested containers follow the same rulesIndented or progressively disclosed content creates a new container context, but it still uses the same grid, gutter, and spacing rules within its available width.

Structural consistency is preservedNested content should align and size fields with the same span system rather than introducing a separate layout model.

Grid Model

The 12-column grid is the structural base for panels, forms, and other layout surfaces. Responsive presentations may simplify to fewer visible groupings, but placement continues to align to this shared grid.

Columns remain fixed to the shared grid. Gutters define the spacing between columns, and the same structure continues to apply as the visible grouping compresses at narrower widths.

Grid Rules

Columns are structuralColumns define placement capacity and alignment. They are not a prescription for how many fields must appear on a row.

Gutters remain consistentThe same 16px gutter is used between columns so spacing stays stable across surfaces and nested containers.

Breakpoints change presentationResponsive changes adjust visible grouping while preserving the same underlying grid logic.

Breakpoints and Containers

Breakpoints describe how the shared grid can be presented across wider and narrower viewports. Container widths define the maximum working width for panel-contained form surfaces.

4 visual columns

Desktop

At larger widths, the system can support up to four visual columns while remaining aligned to the same 12-column base grid.

2 visual columns

Tablet

At medium widths, the same structure can compress to two visual columns without changing the underlying span system.

1 visual column

Mobile

At narrow widths, content may collapse to a single visual column while preserving grid alignment and span logic within the available container.

Container Width

Use 1100px as the maximum layout-controlled panel container width for larger form surfaces. Layout owns that constraint; the panel component remains width-agnostic.

Field Span System

Field widths are expressed as spans within the 12-column grid. The system supports spans of 3, 4, 6, and 12. Use span 4 as the standard standalone field width, span 6 for longer paired or grouped fields, span 3 for compact structured inputs, and span 12 for full-row content.
