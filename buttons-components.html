Buttons

The current system uses four promoted button types: Primary, Secondary, Link, and Destructive. Neutral cancel actions are shown as a grouped-action treatment, not as a fifth promoted button type.

Button Types

Primary carries the main action. Secondary supports adjacent actions. Link buttons stay lightweight inside headers, rows, and compact workflows. Destructive buttons are reserved for irreversible actions.

Primary Button

Use for the main confirm or save action in the current view.

Secondary Button

Use for lower-emphasis companion actions that still need button treatment.

Link Button

Use for lightweight utilities, inline actions, and panel-header actions.

Destructive Button

Use only for permanent actions such as delete, remove, or clear.

Link Buttons With Icons

Link buttons may include a leading or trailing icon. Icons inherit the link color and use the same hover, active, and focus states as the text.

Button Groupings

Use grouped actions to show hierarchy clearly. Neutral cancel stays visually quiet next to a stronger primary or destructive action.

Overview

Buttons communicate action priority. Primary carries the main action, Secondary supports nearby actions, Link keeps utilities light, and Destructive is reserved for irreversible operations. Neutral cancel treatment is used only inside grouped actions.

Anatomy

A button may include an optional icon, a text label, a stateful container, and the shared focus ring. Link buttons support leading or trailing icons without becoming a separate icon-only button type.

Primary button shown with optional leading icon and the standard focus ring.

Parts

Container

36px minimum height, 4px radius, and state-specific background.

Label

14px semibold text with a single concise action verb or phrase.

Optional icon

16px icon that inherits the text color and sits 8px from the label.

Focus ring

Shared blue focus treatment applied consistently across all interactive controls.

Usage Rules

  • Use only one primary button in a local action group.
  • Use destructive buttons only for irreversible actions.
  • Keep link buttons lightweight and avoid using them as the main confirm action.
  • Use the neutral cancel treatment only in grouped actions, not as a promoted standalone button type.
  • Do not create icon-only button variants from this pattern.
