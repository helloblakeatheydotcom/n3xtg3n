Inline Notices

Overview

Inline notices are designed to sit inside panels and support the user’s current workflow. They should stay concise, scannable, and visually secondary to the primary content around them.

Anatomy

An inline notice contains a left icon, a title, and supporting body copy inside a bordered container. In the current direction, the container uses a white surface with a colored border and variant-colored title/icon. Info notices may also include an optional expandable details area.

Container

White surface with a 2px variant border. Success keeps a soft shadow; the others remain flatter.

Icon

20px leading icon that communicates the notice type.

Title

Short status label that anchors the message.

Body text

Supportive instructional or confirmation copy.

Optional details

Info notices may include a compact show/hide control that expands longer guidance below the primary message.

Usage

Use inline notices for contextual feedback inside a panel. Reserve stronger page-level feedback, like snackbar or toast patterns, for cross-page or save-confirmation events.

  • Use info notices for contextual guidance that supports the current task.
  • Use warning notices for issues that need attention soon but do not fully block progress.
  • Use success notices for local confirmation inside the current workflow.
  • Use error notices when the user must resolve an issue inside the current panel before continuing.
  • Keep the title concise and the body copy directly tied to the task in front of the user.
  • Use expandable details only when the notice needs longer supporting guidance that should stay secondary in the collapsed state.
  • Place notices inside panels rather than above the page shell.
