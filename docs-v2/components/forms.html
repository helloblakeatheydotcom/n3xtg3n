Fields

Use fields for structured text entry, single-select and multi-select choices, helper text, validation, and view-only states inside panels and longer process forms.

Overview

Fields provide the shared entry patterns for text controls, selections, helper text, validation, placeholder text, and view-only states. They should feel consistent whether they appear in compact forms, long process pages, or grouped panels.

Core Structure

Label firstEvery field needs a clear visible label unless an equivalent accessible name is provided elsewhere.

Shared support regionHelper text and error text occupy the same space below the control so the rhythm stays stable.

Two main familiesText-based fields handle direct entry, while choice-based fields guide users through single-select and multi-select decisions.

Consistent statesHover, focus, error, disabled, and managed-required states should read predictably across all controls.

Usage Pattern

Text entry stays lightweightUse direct-entry controls for names, dates, short values, formatted inputs, and longer narrative content.

Choice fields stay groupedCheckboxes and radios should be presented as labeled groups with clear option structure.

Validation stays localField errors should explain what needs correction at the point of entry rather than relying on distant page-level messages.

Managed values stay visibleWhen a required value is controlled elsewhere, keep it visible here and direct the user to the source of truth.

Behavior

Field behavior should stay predictable across input types so users can move between text entry and choices without relearning validation, focus, or support patterns.

Text Field Behavior

Hover strengthens the border and lightly tints the field surface.

Focus uses a visible focus ring without changing the overall field shape.

Validation can occur on blur, and invalid fields should set aria-invalid and show a local error message.

Managed-required fields keep the disabled surface while still showing required-state feedback and a next-step link.

Choice Field Behavior

Checkbox groups support multi-select input, while radio groups support exactly one selected option.

Option hover should reinforce the label and control without introducing a separate card-like selection pattern.

Helper text sits above the options, and error treatment should remain attached to the group rather than to an individual choice.

All options must remain keyboard reachable with visible focus and standard checked-state feedback.
