Usage

Buttons define action hierarchy before they define style. Start by choosing the role of the action, then apply the correct treatment.

Action hierarchy

PrimaryMain confirm action in a view.

SecondarySupporting action that still needs button treatment.

LinkLightweight or inline action, not the main confirm action.

DestructiveUse only for irreversible actions.

CancelNeutral button used only inside grouped actions, not as a promoted standalone action.

Grouped actions

  • Use one primary action per group.
  • Place cancel first, followed by the confirming action.
  • Pair destructive actions with a neutral cancel.
  • Keep grouped actions aligned and consistently spaced.

Button Types

Use the correct treatment for the role of the action, then apply the shared states and sizing rules below.

Primary Button

Secondary Button

Size Variants

Use md for standard form, panel-footer, and primary workflow actions. Use sm for compact contexts like panel headers, row actions, and table controls.

Medium (md)

Default size for standard actions.

Small (sm)

Compact size for dense UI.

Patterns

These examples show common button compositions used across the product UI.

Link Buttons With Icons

Link buttons may include a leading or trailing icon. Icons inherit the link treatment and stay secondary to the action label.

Button Groupings

Grouped actions make hierarchy explicit and keep confirming actions aligned with their neutral companion.

Anatomy

A button includes a container, a text label, an optional icon, and the shared focus treatment. Link buttons may use leading or trailing icons without becoming a separate icon-only pattern.

Compact primary button shown with optional leading icon and the standard filled-control focus ring.

Parts

Container

Two sizes: md for standard actions and sm for compact actions.

Label

Semibold action text using the shared button type styles.

Optional icon

Icons inherit the text color and scale with button size.

Focus ring
