Badges and Tags

Use badges for category or type, and tags for status or state. These treatments stay compact, secondary to the record title, and consistent across cards, rows, and related record surfaces.

Category vs status Single badge, single tag, or one of each Guidance, anatomy, token reference

Overview

Badges and tags are compact metadata treatments used beside record content in surfaces like cards and record rows. They are not interchangeable: badges identify category, type, or delivery model, while tags identify status or state.

Core Structure

Badge = categoryUse badges for content type, delivery model, or record category.

Tag = statusUse tags only for state or status such as Inactive or Waived.

Compact and secondaryBoth treatments should remain visually secondary to the record title and primary content.

Shared across surfacesBadges and tags should behave consistently wherever records appear in the product.

Usage Pattern

One badge maxA record may show one badge, but should not stack multiple category badges.

One tag maxA record may show one tag, but should not stack multiple state tags.

Combined pattern is allowedA record may show one badge and one tag together when both category and state are needed.

Divider only when pairedWhen a badge and tag appear together, separate them with a thin divider so the two roles stay distinct.

Guidance

Use badges and tags to support scanning, not to replace the record title or create a second competing hierarchy.

How To Use Badges and Tags

Use badges when the user needs fast category recognition such as service type, assessment family, or delivery model.

Use tags when the user needs fast status recognition such as inactive or waived state.

Use the combined pattern only when both category and status materially improve scanning.

Keep these treatments near the record identity so they read as metadata attached to the item.

Rules

A record may show one badge, one tag, one of each, or neither.

Do not place multiple badges on a single record.

Do not place multiple tags on a single record.

When both appear together, separate them with a thin divider.

Badges vs Tags

Use badges for category or type. Use tags for status or state. They can appear together on a record, but they do not replace one another.

Do not stack multiple badges or multiple tags on a single record. A record may show one badge, one tag, one of each, or neither.

Anatomy

A badge is a rounded category pill. A tag is a square-corner state label with a colored dot. When both appear together, separate them with a thin divider so the two roles stay distinct.

Token Reference

Badges and tags reuse shared typography and color sources. Category badge pairs come from decorative color tokens, while status tags reuse shared state colors already applied in components like Record Rows.
