Migration Rules

These rules keep the migration reversible and low-risk.

  • Do not replace or rename current docs pages during the v2 build.
  • Build new page shells first, then migrate one component at a time.
  • Reuse existing component CSS and tokens where possible; only the documentation chrome should change at first.
  • Cut over the main index only after the v2 pages are materially complete.
