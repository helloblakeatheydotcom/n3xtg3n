Color

Defines the primitive scales and semantic color roles used across the product UI. Component-level aliases live on their respective component pages.

Primitives Semantic tokens

Overview

This page documents primitives first and semantic tokens second. The system uses a three-tier model: primitive scales define available colors, semantic tokens assign meaning, and component aliases apply them. Only the first two tiers live here.

Semantic Tokens

These are the shared color roles used across the system. Each token appears once, grouped by the purpose it serves. Component aliases and pattern-specific implementations belong on their component pages.

Surface

Shared surface roles for page chrome, default containers, subtle utilities, disabled controls, and feedback backgrounds.

TokenPrimitivePurposeUsed by
--color-surface-page#FAFBFDApplication background behind content surfaces.App shell, docs shell
--color-surface-default#FFFFFFDefault surface for controls and containers.All components
--color-surface-subtleSlate 50Low-emphasis utility surface.Utility surfaces
--color-surface-disabledGray 50Disabled control background.Disabled controls
--color-surface-infoBlue 100Informational background role.Info states, notices
--color-surface-successEmerald 50Success background role.Success states, notices
--color-surface-warningAmber 50Warning background role.Warning states, notices
--color-surface-dangerRed 100Danger and error background role.Error states, notices

Border

Boundary roles for subtle containers, default controls, stronger structure, dividers, and feedback states.

--color-dividerGray 200Divider between adjacent items.Lists, tables, sections

--color-border-successEmerald 200Success border role.Success states, notices
--color-border-warningAmber 200Warning border role.Warning states
--color-border-dangerRed 300Danger and error border role.Error states, notices
