Typography

Defines the typeface, size scale, and text roles used across the product UI.

Overview

The typography system defines a single typeface, a constrained weight set, and a reusable size scale. Text styles are assigned by role rather than by component, allowing consistent usage across panels, forms, tables, and other UI structures.

Typeface and Weights

The system uses one primary UI typeface and a limited set of active weights. Keeping the set narrow helps maintain visual consistency across product surfaces.

Primary UI Typeface

Inter

Used for body text, titles, controls, supporting text, tables, panels, cards, sections, forms, and buttons.

"Inter", system-ui, -apple-system, sans-serif

Active Weight Set

  • 400 is used for standard body copy and field text.
  • 500 is used for compact labels and supporting emphasis.
  • 600 is used for titles, labels, badges, and buttons.

No additional product weights are formalized at this time.

Headings

Heading styles are defined by size and emphasis, not by component. The current system uses four composite heading tokens for page, panel, section, and compact content headings.
