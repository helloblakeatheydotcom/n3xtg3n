Choice Controls

Use choice controls for grouped single-select and multi-select decisions. This page covers checkbox groups, radio groups, grouped helper text, validation, and state behavior for selection-based inputs.

Overview

Choice controls provide the shared patterns for grouped single-select and multi-select decisions. They should feel consistent whether they appear inside long forms, panel content, or compact confirmation flows.

Core Structure

Group label firstEvery checkbox or radio group needs a clear visible label that explains the decision users are making.

Shared support regionHelper text and error text stay attached to the group so the rhythm and validation message remain stable.

Two main control typesCheckbox groups support multi-select input, while radio groups support exactly one selected option.

Consistent statesHover, focus, selected, error, and disabled states should read predictably across all options in the group.

Usage Pattern

Use choice for guided selectionChoice controls work best when users should select from known options rather than enter a new value.

Keep options groupedCheckboxes and radios should stay grouped with clear option labels and optional descriptions where needed.

Validation stays localErrors should explain what needs correction at the group level rather than relying on distant page-level messages.

Descriptions stay optionalUse option descriptions only when they add meaning that the label alone cannot provide.

Behavior

Choice control behavior should stay predictable across checkbox and radio groups so users can move between selection patterns without relearning validation, focus, or support rules.

Choice Control Behavior

Checkbox groups support multi-select input, while radio groups support exactly one selected option.

Option hover should reinforce the label and control without introducing a separate card-like selection pattern.

Helper text sits above the options, and error treatment should remain attached to the group rather than to an individual choice.

All options must remain keyboard reachable with visible focus and standard checked-state feedback.
