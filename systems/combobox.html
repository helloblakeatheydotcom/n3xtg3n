Combobox

Combobox is the designed, searchable selection control for option sets that are too long to scan comfortably with Select, need richer row content, or require controlled popup styling.

Searchable selection Single or multiple Optional grouped rows

Overview

Combobox supports searchable selection from a constrained set of options. It can be configured for single selection or multiple selection, with optional features such as secondary text, grouped options, clear, and create-new behavior.

Use Combobox When

The list is too long to scan quicklyUsers benefit from typing to narrow a constrained option set.

The popup rows need system stylingUse Combobox when rows need secondary text, grouping, selected-state treatment, or custom actions.

The workflow needs richer selection behaviorClear, grouped, multi-select, and create-new behavior all fit inside the Combobox family.

Use Something Else When

The list is short and native is acceptableUse Select for short lists where browser and mobile menu behavior is preferred.

The value is unconstrainedUse Text Input when users are not choosing from a system-defined list.

All options should remain visibleUse visible choice patterns when users need to compare a small set of options without opening a popup or searching.

Component Model

Combobox includes two selection modes and a set of optional presentation and interaction features. Start with the selection mode, then add only the supporting features the workflow needs.

Single-Select

Use for one chosen value. The field shows the selected item and can optionally support clear and create-new behavior.

Multi-Select

Use when users need to choose more than one item. Selected values remain visible inside the field as removable chips.

Optional Features

Secondary text, grouped options, clear, selected-state checkmarks, and create-new behavior can be added as needed without changing the core interaction model.

Option Rows

Rows can include a primary label, optional secondary text, grouped sections, selected-state treatment, and contextual actions such as remove for user-added values.

Optional Features

Use optional features to clarify dense option sets or support richer workflows while keeping the core combobox interaction stable and predictable.

Secondary Text

Add a second line when the primary label alone is not enough, such as a person's title or role.

Grouped Options

Use group headers when users need clear visual segmentation across categories or access scopes.

Create and Remove Added Options

Allow create-new only when the product intentionally supports extending the option set. Removable added items should use the same close icon language as the clear affordance.

Clear and Selected State

Use the clear action for current selection only. Use background treatment and a checkmark in the popup to show which option is selected.

Behavior

Combobox behavior should stay predictable across configurations even when optional features are enabled.

Keyboard

Open and searchFocus opens the popup and places the cursor in the search field so typing narrows results immediately.

Move and commitArrow keys move through options, Enter selects, Escape closes, and Tab leaves the control.

Multi-select flowMulti-select should keep the popup open long enough for several selections before the user exits intentionally.

Selection Rules

Single-selectSelecting an option updates the field value, clears the active search query, and closes the popup.

Multi-selectSelected values remain visible as chips inside the field and can be removed individually.

Create-newOnly show the add action when the typed value does not already exist and the workflow explicitly permits custom entries.
