Formatted Inputs

Formatted Inputs use the same field chrome as Text Input while adding value-specific parsing, keyboard, masking, picker, or validation behavior for dates, time, numbers, phone numbers, and similar structured values.

DateTimeNumberPhone

Overview

Use Formatted Inputs when the user enters a known value shape. Keep the visual field treatment consistent with Text Input, then document the additional format rules that make the value valid.

Shared Behavior

Same field anatomyFormatted values still use label, control, helper text, and error message.

Value-specific validationEach format owns its parsing, masking, min/max, and error message guidance.

Mobile keyboards matterUse input types and attributes that support the expected device keyboard.

Format Rules

Each formatted input should define accepted value shape, validation timing, error copy, keyboard behavior, and whether a picker is available.

Date

Document range rules, required date order, manual entry format, and picker behavior.

Time

Document local time assumptions, step increments, and whether typed entry or picker selection is preferred.

Number

Document min, max, precision, step increment, units, and whether negative values are allowed.

Phone

Document country assumptions, mask behavior, accepted digits, and error copy for incomplete numbers.
