Form Controls

Form Controls define the reusable controls users interact with to enter, select, confirm, and submit information. Controls are organized by interaction model: text entry, formatted entry, selection, boolean state, and specialized capture.

Text entry Selection Boolean state Specialized capture

Overview

Use this section to choose the right control, understand its behavior, and apply consistent validation, accessibility, and layout rules across process pages.

Model

Base controlThe reusable component surface and shared anatomy, such as Text Input or Text Area.

Value formatThe data type, parsing, keyboard, masking, and validation rules applied to a base control.

Interaction modelThe way a user provides a value: direct entry, selection, boolean state, or specialized capture.

Layout

Fields stay inside panelsProcess-page form controls should be grouped inside panels with related content.

Widths use LayoutControls participate in the shared Layout foundation spans instead of defining one-off widths.

Patterns decide compositionUse Form Layout for stacked, paired, and dense structured field arrangements.

Choosing the Right Control

Start with the user task. Use direct entry when users know the value, selection when valid values are constrained, and specialized controls only when the interaction requires additional behavior.

Direct Entry

Use Text Input, Text Area, or Formatted Inputs when the user enters a value directly.

Selection

Use Select for simple native lists. Use Combobox when filtering or designed option presentation is required.

Boolean State

Use Toggle when a setting changes immediately and has a clear on/off state.

Specialized Capture

Use Electronic Signature when the workflow requires signature capture, consent, or audit-ready confirmation.

Control Categories

Choose a control category by the kind of user action required, then choose the specific control or formatted variant inside that category.

Text Input

Single-line direct text entry and the base surface for many formatted values.

Text Area

Multi-line narrative entry for longer user-provided content.

Formatted Inputs

Date, time, number, phone, and other constrained value formats.

Select

Native single-value selection for short, simple option lists.

Combobox

Typed filtering and controlled option presentation for searchable lists.

Choice Controls

Checkbox and radio groups for explicit multi-select and single-select choices.

Segmented Control

Compact mutually exclusive choices for modes, filters, or views.

Toggle

Immediate on/off state changes with a visible binary value.

Electronic Signature

Specialized capture for signing, consent, and required confirmation.

Shared Field Anatomy

Field-shaped controls use the same structural parts so labels, help text, validation, and disabled states behave consistently across Text Input, Text Area, Select, Combobox, and formatted inputs.

Required Parts

LabelIdentifies the value and communicates required, optional, read-only, or managed status when needed.

Control surfaceThe interactive field surface for typing, selecting, filtering, or entering a formatted value.

Validation messageError text replaces helper text when the field is invalid and explains what must be corrected.

Optional Parts

Helper textProvides short guidance before validation is triggered.

Status iconUsed when the control needs a persistent visual cue, such as read-only or managed-required.

Next-step linkUsed for managed-required fields when the value must be corrected somewhere else.
