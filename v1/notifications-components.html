Overview

Inline notices are designed to sit inside panels and support the user’s current workflow. They should stay concise, scannable, and visually supportive of the surrounding content rather than behaving like a separate banner system.

Anatomy

An inline notice contains a left icon and supporting message content inside a lightly tinted container. Most notices use a short title plus body copy, but a titleless version is also allowed when a single concise sentence is enough.

Container

Lightly tinted surface with a 1px variant border.

Icon

16px leading icon that communicates the notice type.

Title

Short directive or state label that anchors the message. This is optional when the notice can be expressed clearly in one line.

Body text

Supportive instructional or confirmation copy that expands on the title.

Usage

Use inline notices for contextual feedback inside a panel. Reserve stronger page-level feedback, like snackbar or toast patterns, for cross-page or save-confirmation events.

  • Use info notices for contextual guidance that supports the current task.
  • Use warning notices for issues that need attention soon but do not fully block progress.
  • Use success notices for local confirmation inside the current workflow.
  • Use error notices when the user must resolve an issue inside the current panel before continuing.
  • Keep the title concise and the body copy directly tied to the task in front of the user.
  • Omit the title only when the notice can stay clear and scannable as a single short sentence.
  • Place notices inside panels rather than above the page shell.
