Form Overview

Form components provide the shared entry patterns for text fields, selection controls, helper text, validation, placeholder text, and view-only states. They are designed to support both direct data entry and more structured workflows inside panels and full-page forms.

Parts

Label

Required for clear identification and should communicate required or optional status when needed.

Input control

The primary interactive region for text entry, selection, or choice input, including placeholder text when no value has been entered.

Helper text

Optional; gives supporting guidance before an error is introduced.

Error message

Optional; appears when validation fails, replaces helper text, and should clearly explain what needs correction.

Read-only alert state

Optional; communicates that a required value is missing but cannot be entered in the current context.

Choice options

For checkbox and radio groups, helper text sits above the options and each option includes a control, label, and optional description.

Variants and Optional Elements

Supported Patterns

  • Text input
  • Text area
  • Date and time fields
  • Select list
  • Number with stepper
  • Checkbox group
  • Radio group

Optional Elements

  • Helper text
  • Error message
  • Required and optional indicators
  • Placeholder text
  • Descriptions inside checkbox or radio options
  • View-only icons and read-only alert states

Specifications

Structure

  • Each field should include a visible label unless an equivalent accessible name is provided elsewhere.
  • Helper text and error text share the same supporting region below the control.
  • Placeholder text is secondary guidance only and should not replace labels or helper text.
  • Checkbox and radio groups should be presented as grouped options with clear labels and optional descriptions.

Sizing

  • Single-line fields use a 40px control height.
  • Field labels use 14px type at 600 weight, and entered field values use 14px type at 400 weight.
  • Helper and error text use 13px type.
  • Placeholder text uses a lighter weight and lower-contrast neutral color than entered field values.
  • Choice controls use a 16px control size with consistent option spacing.

Behavior

  • Hover applies a tinted background and accent border for standard entry controls.
  • Focus increases border emphasis and should preserve clear contrast.
  • Error states use the danger border and replace helper text with the error message.
  • Read-only alert states use a softer warning palette that stays distinct from editable error states.
  • Interactive demos may validate on blur and update live while correcting errors.

Accessibility

  • Use semantic label associations for all text-entry controls.
  • Required fields should announce validation errors with `aria-invalid` and live error text where appropriate.
  • Checkbox and radio groups should remain keyboard accessible and preserve visible focus states.
  • Read-only alert states should signal missing required information without using the same treatment as an editable error state.
  • Placeholder text should remain legible but visually secondary to entered values.
