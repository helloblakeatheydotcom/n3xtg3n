Overview

Fields provide the shared entry patterns for text controls, helper text, validation, placeholder text, and view-only states. They should feel consistent whether they appear in compact forms, long process pages, or grouped panels, and they should size within the shared span system defined in Foundations / Layout.

Core Structure

Label firstEvery field needs a clear visible label unless an equivalent accessible name is provided elsewhere.

Shared support regionHelper text and error text occupy the same space below the control so the rhythm stays stable.

Text entry stays primaryUse text-based fields for names, dates, structured values, formatted inputs, and longer narrative content.

Consistent statesHover, focus, error, disabled, and managed-required states should read predictably across all controls.

Usage Pattern

Text entry stays lightweightUse direct-entry controls for names, dates, short values, formatted inputs, and longer narrative content.

Selection stays explicitUse a select list, date picker, or other dedicated field pattern when users choose from a constrained value set.

Validation stays localField errors should explain what needs correction at the point of entry rather than relying on distant page-level messages.

Managed values stay visibleWhen a required value is controlled elsewhere, keep it visible here and direct the user to the source of truth.

Width stays sharedFields support the shared layout spans of 3, 4, 6, and 12. Use Layout to define the span system and patterns to decide when each span is appropriate.

Behavior

Field behavior should stay predictable across text-entry input types so users can move between direct-entry controls without relearning validation, focus, or support patterns.

Text Field Behavior

Hover strengthens the border and lightly tints the field surface.

Focus uses a visible focus ring without changing the overall field shape.

Validation can occur on blur, and invalid fields should set aria-invalid and show a local error message.

Managed-required fields keep the disabled surface while still showing required-state feedback and a next-step link.

Formatted fields should preserve the same field chrome even when the value format changes, such as dates, time, or phone numbers.
